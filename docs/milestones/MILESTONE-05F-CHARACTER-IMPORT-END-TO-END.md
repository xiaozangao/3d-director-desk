# 目标 5F：外部人物导入端到端收口

完成日期：2026-07-17

## 修复内容

- 导入弹窗现在使用统一、可测试的人物准入规则。
- 没有蒙皮骨架的文件只能作为静态道具加入，不再出现误导性的“作为人物加入”。
- 需要手动映射的人物在确认时再次校验 15 个身体部位，不能通过界面状态或调用顺序绕过。
- 带可播放自有动作的人物导入后自动选择第一段动作，从 0 秒开始预览。
- 兼容人形导入后依次自动预览走路、跑步、跳跃和挥手，每项显示当前名称与 `1/4` 进度，可随时跳过。
- 四项自检使用纯运行时动作覆盖，不修改角色正式动作、不写入工程 JSON、不产生撤销记录；结束或跳过后恢复导入前的播放位置和暂停状态。
- `native-only` 未知骨架只轮播自身有效 clip，不会强套标准人形动作。
- 通用 GLB 没有匹配的自带 clip 时，可回退到外部标准动作；已有匹配自带 clip 的 X Bot、Soldier、RobotExpressive 仍优先使用自身动作。
- 自带动作资产记录来源人物资产 ID。
- `native-only` 人物的动作页现在能继续选择自己的自带动作，但不会显示标准人形预设或其他未知骨架动作。
- 旧工程没有来源 ID 时，通过相同 URL 和文件名继续识别自带动作。
