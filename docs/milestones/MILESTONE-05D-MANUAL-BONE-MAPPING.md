# 目标 5D：外部人物手动骨架映射

完成日期：2026-07-17

## 完成内容

- 人物体检新增“骨骼名列表 + 自动骨骼映射”结果，会自动匹配 Mixamo、mixamorig1、Bip/UE4、CC Base 等已知命名。
- 体检结果为“需要手动映射”时，弹窗显示 15 个主要身体部位的下拉选择。已识别部位会预选，用户只补空项。
- 必须映射完整且每个部位使用不同骨骼才能“作为人物加入”，避免同一根骨骼被误配到多个部位。
- 骨架映射作为人物资产元数据持久化，并传入主视口、成片监看、第一视角和导出场景的同一个运行时结构。
- 身体部位跟拍会优先读取存储的映射，不再只靠骨骼名猜测。
- Mixamo 外部动作的轨道会按语义身体部位重定向到用户选定的目标骨骼。
- 完整映射的 Bip、CC Base、通用人形可使用可识别的外部动作；无法识别的动作文件仍会被拦截。

## 回归保护

- Mixamo 轨道可重定向至任意手动命名的目标骨骼。
- 语义跟拍优先使用保存的映射。
- 完整映射要求 15 个必需部位且骨骼不重复。
- UI 覆盖：通用骨架 -> 用户选择 -> 作为人物导入 -> 映射写入资产。
- 完整映射人物与 Mixamo、CC Base 动作 profile 兼容；未映射的 Bip 与 Mixamo 仍不兼容。
