# 目标 3：人物与镜头统一自定义段内节奏

完成日期：2026-07-17

## 本阶段完成

- 人物路线与镜头路线已共用 `routeTiming`：平滑/折线、按弧长匀速、整条路线柔和、经过/停留、停留秒数和自定义到达时间均使用同一计时内核。
- 补齐此前只有数据字段、没有用户入口的 `customEasing`。
- 人物和镜头路线共用同一个控件，提供直线、慢起、慢停、两头柔和四种段内节奏。
- 四种曲线分别为 `[0,0,1,1]`、`[0.42,0,1,1]`、`[0,0,0.58,1]`、`[0.42,0,0.58,1]`。
- 控件只在“自定义”速度模式出现；匀速和柔和继续按整条路线自动计算。
- 自定义模式保留每个中间点的单独到达时间；起点固定 0 秒、终点固定总时长。
- 曲线直接写入现有工程 JSON，不引入第二套路线格式。

## 验证

- 预设映射、持久曲线识别和未知曲线回退测试通过。
- 共用 `routeTiming` 验证速度曲线会改变真实采样位置，而不只是切换按钮状态。

- 浏览器隔离实例中，镜头选择“慢起”、人物选择“两头柔和”，两处均唯一 `aria-pressed=true`。
- 镜头和人物各生成 2 个真实路线点；页面横向溢出为 0，业务控制台错误为 0。
