# 目标 7C：三档性能场景、动态自动档与匿名报告

完成日期：2026-07-16

## 用户功能

- “性能”面板提供轻量、中等、重型三个标准测试入口，新标签使用临时工作区，不覆盖当前导演台。
- 基准页显示预热、采样和完成状态；完成后可手动下载匿名 JSON 报告。
- 自动档不再只看 CPU 线程和内存。它会读取主 3D Canvas 的真实渲染帧，在均衡与流畅之间动态调整。
- 自动档连续两个差窗口才降档，连续三个好窗口才恢复，并有 9 秒冷却，避免来回切换。
- 自动切档只热更新主视口、监看和 Gizmo 的 DPR，不因切档重建 WebGL Canvas。
- 自动档上限为内部均衡；高清保留为用户手动选择。
- 基准、参考视频导出、后台标签、系统休眠产生的超长帧不参与自动档判断。

## 固定场景

| 场景 | 人物 | 道具 | 成片监看 | 全景 | 人物路线 |
| --- | ---: | ---: | --- | --- | --- |
| 轻量 | 2 | 5 | 关闭 | 关闭 | 2 条 |
| 中等 | 5 | 20 | 开启 | 关闭 | 5 条 |
| 重型 | 10 | 50 | 开启 | 开启 | 10 条 |

三档共用三点循环镜头、人物动作和人物路线。重型场景的 50 个道具位置互不重叠，全景使用仓库内固定的 2048×1024 测试图，保证重复测试可比。

## macOS 浏览器实测

测试设备：Apple M5 Pro，Chrome 150，1280×650 CSS 主视口。

| 场景 / 档位 | 平均 FPS | 1% Low | P95 帧时间 | 主画布像素 |
| --- | ---: | ---: | ---: | --- |
| 轻量 / 流畅 | 73.2 | 40.2 | 17.5 ms | 960×487 |
| 中等 / 均衡 | 67.7 | 40.0 | 17.8 ms | 1280×650 |
| 重型 / 高清 | 64.3 | 39.7 | 24.7 ms | 2560×1300 |

这些数字只代表当前机器和内置浏览器环境，不代表 Windows 结果。浏览器调度可能出现高于显示器刷新率的统计值，跨机器比较时应同时看 1% Low 和 P95 帧时间。

## 匿名报告边界

下载文件固定名为 `director-performance-report.json`，通过逐字段白名单构造，只包含：

- 平均 FPS、1% Low。
- 平均、P50、P95、P99 帧时间。
- 浏览器家族和主版本、WebGL GPU 名称、逻辑线程数。
- 标准化操作系统类型：Windows、macOS、Linux、ChromeOS、Android、iOS 或未知系统。
- 人物/道具数量、监看和全景开关。
- CSS 画布尺寸和实际像素尺寸。

不包含用户名、时间戳、导演台 ID、工程名、模型名、URL、Cookie、IP、本地路径、完整 UA、原始平台字符串、工程 JSON 或任何场景内容。

公开报告 schema 已升级为 v3：使用标准化操作系统类型区分 Windows 与 macOS，并增加测试场景、性能档和软件版本，使群友报告具备可比较上下文。

## 配置建议

这是场景规模建议，不是尚未完成的 Windows 实机认证：

| 级别 | 建议硬件 | 建议档位 | 适合场景 |
| --- | --- | --- | --- |
| 最低 | 4 核 CPU、8 GB 内存、可用 WebGL2 的集显 | 流畅 | 轻量，少量人物与道具，不长期打开监看 |
| 推荐 | i5-13400 或同级、RTX 3060、16 GB 内存 | 均衡 | 中等，5 人 / 20 道具并开启监看 |
| 适合重型 | 8 核以上 CPU、RTX 4060 / 3070 或同级、32 GB 内存 | 自动或高清 | 重型，全景、10 人 / 50 道具、多路线 |

Windows 用户应优先使用 Chrome 或 Edge，并通过匿名报告补充真实数据。`i5-13400 + RTX 3060 + 16 GB` 是中等场景的目标推荐配置，目前没有 Windows 真机结论。
