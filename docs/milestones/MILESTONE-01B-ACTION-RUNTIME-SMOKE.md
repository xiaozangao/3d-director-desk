# 目标 1B：动作跨视图真实运行时验收

完成日期：2026-07-17

## 根因与修复

- 真实 `camille.fbx + wave.fbx` 在首帧能显示动作，但稳定后回到绑定姿势。
- `MixamoAnimationPlayer` 每帧先恢复绑定姿势，再对同一时间重复调用 `AnimationMixer.setTime()`。Three.js 会缓存上次混合结果，相同时间不再重写骨骼，因此只剩绑定姿势。
- 新增绝对时间采样函数，记录上次已应用的片段时间。时间未变时保留当前骨骼姿势；只有时间改变时才从静止基线重新采样。
- 新增回归测试，覆盖“后续帧重复采样同一时间时，动作姿势不得消失”。

## 独立验收入口

- 入口：`../../examples/experiments/action-runtime-smoke.html`
- 真实资产：`camille.fbx` 人物与 `wave.fbx` 动作。
- 视图：导演主视口、成片监看小窗、第一视角、看成片、视频导出。
- 时间点：0%、25%、50%、75%、100%。

- 生命周期：倒拖到 50%、重播到 0%、重播到 50%。
