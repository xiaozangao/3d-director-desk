# 目标 8C：插件结果回传与宿主客户端

完成日期：2026-07-16

## 本阶段交付

- 工程快照新增确定性 `projectFingerprint`，插件可把读取时的工程版本绑定到结果。
- 二创协议新增：
  - `plugin.result.submit`：提交受限结构化插件结果。
  - `plugin.results.list`：查询当前页面运行期间最近 50 条结果。
- 每次查询都会按当前工程指纹重新计算 `stale`：用户在插件计算期间修改工程后，旧结果自动标记过期。
- 插件结果包含插件 ID、名称、版本、结果类型、状态、摘要、JSON 数据、接收时间和结果 ID。
- 结果收件箱只存在当前页面运行时，不写工程 JSON、不上传服务器、不自动修改场景。
- 提交结果会触发 `storyai:director-desk-plugin-result` 本地 CustomEvent，供未来界面或二创模块订阅。

## 安全与资源限制

- `data` 必须是可序列化 JSON，禁止函数、DOM、循环引用和可执行代码。
- 单条结果最多 512 KB，收件箱最多 50 条。
- 插件 ID 只允许字母、数字、点、下划线和连字符。
- 非法结果返回 `invalid-plugin-result`，保留原 requestId。
- v1 不提供自动应用镜头、删除工程或任意代码执行；未来“应用结果”必须另设权限和用户确认。
- 宿主消息继续同时校验 `event.origin` 和 iframe `event.source`。

## 可运行宿主示例

- `examples/infinite-canvas-embed.tsx` 已升级为完整 React client。
- 支持能力、工程、时间轴、帧、视频、提交结果和结果列表方法。
- 每个请求使用 UUID，支持并发 requestId 配对、普通请求 15 秒超时、视频请求 60 秒超时、卸载清理和错误 Promise。
- 严格验证消息 origin、iframe source、协议版本和响应 action；错误 action 不会误解开另一个在途请求。
