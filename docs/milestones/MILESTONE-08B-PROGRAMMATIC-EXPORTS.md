# 目标 8B：程序化成片帧与参考视频导出

完成日期：2026-07-16

## 本阶段交付

- 二创协议新增 `export.frame`：
  - 可选择 `current`、`first`、`last`。
  - 支持 720p / 1080p 清晰度档。
  - 返回 PNG `dataUrl`、文件名、实际宽高、帧位置和归一化进度。
- 二创协议新增 `export.video`：
  - 支持 720p / 1080p 和 24 / 30 / 60 FPS。
  - 返回 WebM `Blob`、MIME、文件名、实际宽高和时长。
  - 协议调用不自动下载，由宿主自行保存、上传或创建下载链接。
- 界面“导出 WebM”继续自动下载，但现在与协议共用同一个 Blob 渲染结果。
- 帧和视频共用隐藏的干净成片 Canvas：不包含 UI、编辑网格、轨迹线和轨迹点，沿用看成片 FOV、当前画幅、全景和场景动作。
- 720p / 1080p 是清晰度边界，实际宽高跟随当前画幅。16:9 的 720p 为 1280 x 720；自动宽屏画幅会按实际比例计算高度。
- 帧/视频导出结束后恢复用户原进度和播放状态，不再强制停在尾帧。
- 同一时间只允许一个媒体导出任务，并发请求返回 `export-busy`；渲染器未就绪、镜头不足或浏览器不支持时返回关联原 requestId 的 `export-failed`。

## 实现边界

- 视频结果使用可结构化克隆的 `Blob` 通过 postMessage 回传，不把数 MB 视频转成 base64。
- PNG 帧使用 `dataUrl`，便于宿主直接预览或保存。
- 导出器返回媒体数据，下载行为移到界面调用层；程序化调用不会产生意外下载。
- 隐藏渲染 Canvas 使用 portal 挂载到 document.body，避免被导演台布局压缩。
- 基准模式支持显式 `benchmarkProgress` / `benchmarkPlaying=paused`，仅用于确定性导出验证，不给正式协议增加远程时间轴写权限。

## 浏览器验证

- 真实 `export.frame`：16:9 / 720p 返回 1280 x 720 PNG，首帧进度为 0。
- 自动画幅测试返回 1280 x 364，符合 iframe 当前约 3.52:1 的实际画幅，并非 CSS 压缩。
- 真实 `export.video`：录制 8 秒 VP9 WebM，Blob 约 2.2 MB，MIME 为 `video/webm;codecs=vp9`。
- 确定性恢复测试：导出前后均为 42% / 3.36 秒 / 暂停 / `cam_1`，完全一致。
- 播放中恢复由自动测试覆盖：先恢复原进度，再恢复播放状态继续前进。
- 2026-07-17 增强 `/extension-video-smoke.html`：宿主收到真实 Blob 后会直接解码 WebM，并抽取 0.1 秒与 5.6 秒两帧做像素分析。
- 最新真实接口结果：VP9 WebM、`2,325,689` bytes、8 秒、1280 x 322；响应宽高与解码宽高完全一致。322 高度来自 iframe 的自动宽画幅。
- 两帧亮度分别为 `0.044977`、`0.061485`，方差分别为 `0.011779`、`0.018417`，均非黑屏；归一化帧差为 `0.075607`，证明视频不是静止帧。
- 导出前后时间轴均为 42% / 3.36 秒 / 暂停 / 导演视角 / `cam_1`，完全一致。
- 浏览器结果截图：`../assets/smoke-results/extension-video-smoke-result.png`。
- 内置浏览器嵌套 iframe 仍有无来源 URL 的 MutationObserver 注入层错误；普通页面与 iframe 业务 Canvas 正常，不来自项目代码。

### 2026-07-17 黑帧修复与最终复验

- 严格真实时间录制发现旧流程在 WebM 最前约 0.1 秒可能先写入黑帧；根因是 MediaRecorder 早于隐藏成片 Canvas 的稳定首帧启动。
- 录制前现等待成片 Canvas 连续完成两帧绘制，MediaRecorder 启动后再等待一帧才开始镜头播放。
- 修复后真实 8 秒 VP9 WebM 为 `621,328` bytes、1280 x 633；0.1 秒首帧亮度 `0.037133`、方差 `0.010747`，不再是黑帧。
- 5.6 秒后段亮度 `0.079405`、方差 `0.014062`；首段与后段帧差 `0.086529`，证明视频持续变化。
- 导出前后均为 42% / 3.36 秒 / 暂停 / 导演视角 / `cam_1`，完全恢复。
- 首帧、当前 42%、尾帧 PNG 也在同一真实 iframe 中连续通过；三帧均非黑，首尾帧差 `0.03642`。
- 最新证据图：`../assets/smoke-results/extension-video-smoke-result-v2.png`、`../assets/smoke-results/extension-protocol-smoke-result-v2.png`。
