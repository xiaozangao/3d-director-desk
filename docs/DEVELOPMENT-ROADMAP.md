# 3D 导演台开发路线图

更新时间：2026-07-17

## 开发纪律

- 每个目标拆成可独立验收的小目标。
- 每个小目标按“实现 -> 自动测试 -> 生产构建 -> 浏览器自验证 -> 检查点文档”完成。
- 完成一个小目标后，以检查点文档作为下一轮的压缩上下文。
- 未经用户明确确认，不提交、不推送、不更新 GitHub 或在线体验版。
- 不回退工作树中无法确认归属的已有改动。

## 实际执行顺序

按用户最后确认的顺序，以稳定目标编号记录为：

`目标 1 -> 目标 2 -> 目标 3 -> 目标 7 -> 目标 4 -> 目标 8 -> 目标 5 -> 目标 6 -> 目标 9 -> 目标 10`

- 目标 9：高斯泼溅格式加载与独立浏览器承载实验。
- 目标 10：高斯空间取点、地面采样、代理碰撞和大场景性能实验。
- 编号只作为稳定引用，后续调整优先级时不再给目标重新编号。

### 目标 2：统一人物与镜头路线

- 镜头和人物路线统一支持平滑/折线、匀速/柔和/自定义。
- 路线点支持经过或停留，并可设置停留秒数和人物停留动作。
- 指定停留动作从到点时刻 0 秒起播，停留结束后恢复下一移动段动作；真实 Camille 五视图和重播验收通过。
- 底部双轨时间轴同时显示镜头与人物的移动段、停留段。
- 匀速模式经过中间点时不自动刹停。
- 自定义模式可选择直线、慢起、慢停和两头柔和的段内节奏，人物与镜头共用同一控件和数据字段。
- 主视口、运镜工作台和摄影机轨迹面板统一按计算后的真实到达时间高亮和跳转，不再误用旧轨迹点时间。

检查点：备份目录中的 `milestones/MILESTONE-02A-ROUTE-TIMING.md`、`02B`、`02C`，以及 `milestones/MILESTONE-02D-ROUTE-HOLD-ACTION-SMOKE.md`、`milestones/MILESTONE-02E-CUSTOM-ROUTE-EASING.md`。

### 目标 5：外部绑骨人物与动作导入

状态：完成，真实非 Mixamo 用户资产的最后体验待补。

## 下一目标

### 目标 7：Windows 性能档位与基准测试

状态：本地开发阶段已完成。macOS 已实测，Windows 真机数据等待群友匿名报告补充，不伪造结果。

小目标计划：

1. 建立同一场景、同一动作、同一镜头的可重复性能基准。已完成，见 `milestones/MILESTONE-07A-PERFORMANCE-BASELINE.md`。
2. 审计主视口、成片监看、模型、纹理和动画的主要性能开销。已完成。
3. 增加自动、流畅、均衡、画质优先四档，小白只需要选档位。已完成。
4. 档位统一控制 DPR、抗锯齿和监看渲染精度，不改变项目内容。已完成。
5. Windows 与 macOS 分别记录基准结果；macOS 已完成，Windows 明确标记为未验证，不伪造数据。
6. 检查低档位下人物动作、镜头跟踪和时间轴仍保持一致，不通过删功能换帧率。已完成 macOS 自动与浏览器验证。
7. 补齐轻量（2 人 / 5 道具）、中等（5 人 / 20 道具 / 监看）和重型（10 人 / 50 道具 / 全景 / 多路线）三种标准场景。已完成。
8. 提供不采集隐私的群友基准报告导出，记录浏览器、显卡、场景规模和 FPS；整理最低、推荐和适合配置。已完成。
9. 匿名报告新增标准化操作系统字段，可区分 Windows、macOS 和 Linux，不导出原始平台字符串。已完成。

检查点：`milestones/MILESTONE-07A-PERFORMANCE-BASELINE.md`、`milestones/MILESTONE-07B-PERFORMANCE-PROFILES.md`、`milestones/MILESTONE-07C-BENCHMARK-SCENES.md`。
